Background

TmCurator is part of a project that aims to generate a high-quality reference dataset of upstream-downstream relationships between proteins in Saccharomyces cerevisiae. In the first phase of the project, raw relationships were extracted from a large corpus of publications using text mining software. The second phase of the project will consist of manually curating these sentences to achieve high quality.

As part of the text mining process, a list of S. cerevisiae gene and protein names and a list of hierarchically arranged potential relationships (e.g. "binds", "interacts", "regulates") were assembled. These lists were used with the GeneWays text mining system, extracted plausible sentences and their contents from the publication corpus.

As part of the manual curation process we are aiming to (i) remove false extractions from the dataset, and (ii) salvage as many inaccurate extractions as possible by correcting the computer's interpretation. To make this process as efficient and time-saving as possible, we have created the TmCurator tool. It collates extracted sentences in pages, each of which concerns one pair of genes or proteins. Each curator is assigned a specific list of 300 pages, such that the list overlaps with those of two other curators. Each page is evaluated by three different persons. Thus, we hope to achieve a high quality of curation.

System requirements

To use TmCurator you will require a HTML5-compatible internet browser, such as

  • Firefox 2.0 or higher
  • Chrome 4.0 or higher
  • Opera 9.0 or higher
  • Safari 3.1 or higher
  • Internet Explorer 9.0 or higher

Using the TmCurator interface

Open your browser and navigate to http://dalai.mshri.on.ca:8081/ to see the TmCurator interface. A dialogue window will be visible, asking you to log in. Assuming you do not yet have an account, click on the "Create new account" button. The dialogue box will change, prompting you to choose a user name and password.

After successfully logging in, you will see the welcome screen. The welcome screen shows a percentage bar indicating your current progress. Underneath you will find two buttons labelled "Continue" and "Jump to Pair". You may also notice a third button in the top right corner of your screen that shows your chosen user name. You can use this button to log out of your session at any time. Unless you do so, the page will remember your user name (using a browser cookie) and automatically log you in whenever you navigate to the page from the same computer.

For the purposes of this tutorial you will only need to use the "Continue" button. Advanced users may read about the "Jump to pair" button in the appendix. Click on the "Continue" button. It will take you to the main curation interface, displaying a page on a particular pair of genes. The names of the two genes are displayed on the top of the page and highlighted in blue and orange. Below this title is a list of boxes. Each of these boxes represents a sentence which has been automatically extracted from a publication, in which the names of the two genes have been highlighted. Grouped around the sentence are a number of information and control elements.

  • The title bar. Each box has a narrow title bar which displays a unique ID for the sentence as well as any important messages. For example, if the sentence in question originates from a publication that was not approved by SGD as related to S. cerevisiae, a red warning message will appear here.
  • The "PubMed" button. Clicking this button will open the publication from which this sentence has been extracted in a new browser tab. This may be useful to find the exact context of the displayed sentence if necessary. You can also hover your mouse over the button for a few seconds, causing citation information for the publication to be displayed.
  • The verdict field. This component displays the computer's current interpretation of the sentence. Two symbols represent the currently assumed entity types for the two partners, i.e. whether they are interpreted to be genes, proteins, transcripts or neither. Between them you will find a labelled line representing the currently assumed relationship between the two entities. If applicable, the line will have an arrow tip indicating the directionality of the relationship.
  • The "Switch" button. This button allows you to switch the direction of the arrow (if applicable).
  • The "Entities" button. This button will let you change the entity types for the two objects in question. Available entity types are "Gene", "Protein", "Transcript" and "Unknown". Upon clicking the button, a dialogue box will open, offering two drop-down menus from which you can select the above entity types.
  • The "Action" button. This button allows you to adjust the action (i.e. relationship) between the two genes or proteins. When you click the button, a new dialogue window will appear. In its centre it contains a tree structure from which you can choose a word that reflects the true relationship between the two entities expressed in the sentence more accurately. Since the list is relatively extensive, a search bar at the top will allow you to find particular word you may be looking for. Finally, for your convenience a collapsible panel at the bottom of the dialogue window can display the ten words used most frequently by all curators. You can reveal the panel by clicking on the small button with double arrow symbol.
  • The "Negate" button. This button can be used to negate a relationship. That is, for example to turn the statement "A interacts with B" into the statement "A does not interact with B". A red cross-out symbol will appear on the arrow in the verdict field to represent the negation.
  • The "Invalid" button. You can use this button to declare an extraction as invalid altogether. That is, to declare that the sentence does not at all make a statement about the two genes in question.
  • The comment box. If you feel that there is a problem with a particular sentence that can not be addressed appropriately using the available options, you can leave a comment in this box, describing the problem.

When you have completed your curation of the sentences on a page you can go to next page for another gene pair by clicking on the "Next" button at the bottom of the page. This will cause all your changes to be saved to the database and the next page to be loaded. If, for some reason, you wish to go back to the previous page, you can use the "Previous" button instead. The "Home" button will bring you back to the start screen.

Exemplary cases

Here we list a couple of exemplary problem cases and offer solutions.

Case 1: A sentence has nothing to do with the two genes or does not at all express a relationship between them. Mark the sentence as invalid using the "Invalid" button.

Case 2: A sentence expresses only hypothesizes that a relationship between the two genes exists or does otherwise not seem like appropriate evidence. Mark the sentence as invalid using the "Invalid" button. You may also choose to enter a description of the problem into the comment box.

Appendix

Reporting errors

If you encounter any bugs, errors or other problems with the software, please submit a report to the issue tracker.

Jumping to custom gene pairs

If for some reason, you wish to find a particular gene pair in the dataset you can do so by going back to the main page and clicking on the "Jump to pair" button. It will open a new dialogue window with a slider. You can use the slider to select a specific page in the data set. You can also search for specific genes by name using the search bar on the top of the dialogue. After submitting a search term, all pages with genes that match your query will be listed in a table below. You can then select your desired page from the list and the slider will jump to the corresponding position. Click "OK" to open the selected page.

Adding new sentences

Another advanced feature within the interface is the option to add new sentences to a gene pair. To do so you can use the "Add sentence" button on the top left of the desired gene pair page. The button opens a new dialogue window which will prompt you to enter the sentence and the PubMed ID of the publication from which the sentence originates. After submitting the sentence it will be displayed at the bottom of the page and can be used like any other pre-existing sentence for that gene pair.
